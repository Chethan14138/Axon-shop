E-Commerce Shopping Platform
Beginner-Level Extended Documentation

1.1 Purpose
This document describes the functional and technical requirements of a Beginner-Level E-Commerce Shopping Platform. It is prepared for academic projects, seminars, and beginner developers to understand how an online shopping system works.

1.2 Scope
The system allows users to browse products, add them to a cart, place orders, and track purchases. Admins can manage products, users, and orders using a dashboard.
 Functional Requirements (Extended – Beginner)
1.	User Features
•	User registration using email and password
•	Login & logout system
•	Forgot password (basic – reset using email or admin help)
•	View profile details (name, email, phone)
•	Update profile information

2.	Product Features

•	View all products on the home page
•	View product details (image, price, description)
•	Filter products by category (clothes, electronics, etc.)
•	Sort products by price (low → high, high → low)
•	Search products by name
•	Cart & Order Features
•	Add product to cart
•	Remove product from cart
•	Increase or decrease quantity
•	View total price
•	Place order after checkout
•	View order history
•	Order status: Placed, Shipped, Delivered
•	
3.	Payment Features 
4.	
•	Cash on Delivery (COD)
•	payment / simulated payment 

5.	Admin Features

•	Admin login
•	Add new product (name, price, image, stock)
•	Edit product details
•	Delete products
•	View all user orders
•	Update order status
•	Check low-stock products
 Non-Functional Requirements (Very Simple)

•	Easy to use UI (beginner-friendly design)
•	Fast page loading (under 3 seconds)
•	Mobile and desktop responsive design
•	Secure login system
•	Password encryption
•	Role-based access (User / Admin)
•	Input validation (avoid empty forms)
 5. System Requirements
 5.1 Hardware Requirements
•	Smartphone or computer
•	Internet connection
5.2 Software Requirements
•	Android / Web browser
•	Backend server
•	Database (MySQL or Firebase)
---
6. Assumptions
•	User has internet access
•	Delivery is done within local area
•	Limited number of products
---
 7. Limitations
•	No real-time GPS tracking
•	Limited payment options
•	No advanced recommendations
---
8. Conclusion
The Grocery Delivery Application provides a simple and effective way for users to order groceries online. This beginner-level system focuses on basic functionality and ease of use.
